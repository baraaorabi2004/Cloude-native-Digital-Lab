تحويل الموارد والأجهزة القديمة في جامعة البلقاء إلى "مختبر رقمي سحابي" يوفر بيئة تدريبية ذكية للأمن السيبراني، مع التركيز على الاستدامة وتقليل النفايات الإلكترونية.

- **Cloud Infrastructure:** Docker & K3s (Kubernetes).
- **AI Intelligence:** Google Gemini 1.5 Flash (AI Lab Tutor).

- **Predictive Resource Management:** توفير الطاقة عبر توزيع الأحمال بذكاء بناءً على أوقات الذروة.

# 🌐 مختبر سيسكو السحابي (Cisco Packet Tracer) - تقرير المشروع 🚀

## 1. وصف المشروع
هذا المشروع عبارة عن بيئة مختبرية افتراضية سحابية مخصصة لطلاب الشبكات. تتيح لهم استخدام برنامج **Cisco Packet Tracer** مباشرة عبر أي متصفح ويب دون الحاجة لتثبيته على أجهزتهم الخاصة، مما يسهل عملية التعلم من أي مكان.

## 2. التقنيات والأدوات المستخدمة
- **الحاويات (Containerization):** استخدمنا Docker مع توزيعة Ubuntu 22.04 كقاعدة للنظام.
- **الوصول عن بُعد:** تقنية **KasmVNC** لتوفير سطح مكتب سريع وسلس عبر المتصفح.
- **بيئة سطح المكتب:** **XFCE**، وهي بيئة خفيفة جداً تضمن كفاءة الأداء وتوفير موارد الذاكرة والمعالج.
- **البنية التحتية السحابية:** Google Cloud Run لاستضافة الخدمة بشكل سحابي كامل.
- **أداة البناء:** Google Cloud Build لبناء صور النظام (Images) مباشرة على السحابة.

## 3. تفاصيل سطح المكتب الافتراضي (KasmVNC)
### ما هو وما نوعه؟
هو خادم VNC حديث يعتمد على تقنيات الويب (Rendering via WebGL/WebSockets). على عكس الـ VNC التقليدي، هذا النوع لا يتطلب تثبيت أي تطبيق على جهاز الطالب، بل يعمل بالكامل داخل **المتصفح**.

### لماذا اخترنا KasmVNC؟
1.  **الأداء العالي:** يوفر سرعة استجابة مذهلة حتى مع الاتصالات الضعيفة.
2.  **السهولة:** الطالب يفتح رابطاً فقط ويبدأ العمل فوراً.
3.  **الأمان:** يدعم التشفير عبر HTTPS بشكل افتراضي.
4.  **التخصيص:** مكننا من تغيير ألوان الواجهة لتناسب هوية مشروعنا (الأزرق السماوي).

### كيف تم تثبيته؟
تم تحميل حزمة الـ `.deb` الخاصة بـ KasmVNC وإدراجها داخل ملف الـ Dockerfile ليتم تثبيتها تلقائياً عند بناء المختبر، مع ضبط إعدادات المستخدم والصلاحيات لضمان حماية النظام.

### استخدامه في المختبر:
يستخدم ليكون "الوسيط" الذي يظهر واجهة برنامج Cisco Packet Tracer للطالب. بدونه، لا يمكن رؤية واجهة البرنامج الرسومية في البيئات السحابية التي لا تملك شاشة.

## 4. خطوات التنفيذ (كيف تم بناء المختبر)
1.  **إعداد البيئة:** قمنا بتكوين نظام لينكس بكافة المكتبات اللازمة لتشغيل الواجهات الرسومية (X11) ومكتبات Qt5.
2.  **دمج Packet Tracer:** تثبيت النسخة 8.x من البرنامج وأتمتة الموافقة على شروط الاستخدام (EULA).
3.  **إصلاح التبعات (Dependencies):** أضفنا مكتبة `libqt5networkauth5` لإصلاح خطأ التشغيل الشهير في Packet Tracer.
4.  **تخصيص الواجهة:** استخدمنا CSS مخصص لتغيير ألوان شاشة الدخول وشريط المهام ليكون باللون الأزرق السماوي.

## 5. الأكواد والأوامر الرئيسية المستخدمة
### بناء المختبر على السحابة (Build)
```cmd
gcloud.cmd builds submit --tag gcr.io/project-9e4d13ce-1fe8-45b6-87b/cisco-lab .
```

### نشر المختبر وتفعيله (Deploy)
```cmd
gcloud.cmd run deploy cisco-lab --image gcr.io/project-9e4d13ce-1fe8-45b6-87b/cisco-lab --platform managed
```
